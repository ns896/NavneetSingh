UDP Server using LWIP without RTOS
Creating a UDP server on STM32H7 using a LWIP stack for communication between a x86PC and the micro controller using the Ethernet/IP stack. Main objectives is to achieve a much higher rate of data transfer and also support PoDL, hence cutting the need for a separate power line when deploying in the felid.

Project Details / Background
The primary motivation for developing on LWIP was to enable Ethernet networking communication capabilities for field-deployed systems. The LWIP network stack comes pre-initialized with CubeIDE, simplifying the setup process. In this example, an RTOS is not used because the microcontroller (MCU) is small and does not necessitate the overhead of an operating system. Instead, the node integrates with a larger processing system, such as an x86 PC or a powerful SoC like the NVIDIA Jetson Xavier AGX, which can handle substantial calculations and processing tasks offloaded from the smaller nodes.
Main topics which are significantly important are covered in the following points:
  • MCU Hardware Setup Between MCU & Ethernet PHY
  • STM32 Memory Configuration for Ethernet stack
  • Configuring MPU in ARM Cortex-M3
  • Configuring the Ethernet GPIOs
  • Configuring the clocks
  • Configuring LWIP
  • Linker list addition of stack for the Ethernet buffers for TX and RX
  • Addition of execution code that creates a server on the STM32 microcontroller
  • Compilation and debugging
